                        Hi,

                        Unfortunately it won’t be amazing.. I’ve tried it with a few thousand features with a couple hundred samples on simulated data and the results were less than impressive.. To be honest, I don’t think there’s any point in doing multivariate feature selection in such p>>n situations.. I’d just go univariate, use mutual information between each column of my X and my y vector. I guess you’re talking about text data.. If so, have a look at PyFEAST, which is a Python wrapper around a bunch of information theoretic feature selection algorithms (FEAST package), implemented for discrete data (bot X and y has to be discrete though)..
